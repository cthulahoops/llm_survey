Prompt

I need to take this json:

{
'first_name': 'Jane',
'last_name': 'Smith',
'date_of_birth': '03/16/1977',
orders: [
{
'created': '2024-01-25T15:00:04',
'amount': '100.00'
}
]}

And generate some dataclasses.

date_of_birth needs to be a date.
created is a datetime.
amount is a decimal.

How would I do this using the marshmallow library in python?

Response 1.

Using Marshmallow to Generate Dataclasses from JSON

Here's how you can achieve this using the marshmallow library in Python:

from marshmallow import Schema, fields, post_load
from marshmallow.validate import Length
from datetime import date, datetime
from decimal import Decimal

class OrderSchema(Schema):
    created = fields.DateTime()
    amount = fields.Decimal(as_string=True)

class CustomerSchema(Schema):
    first_name = fields.Str(required=True, validate=Length(min=1))
    last_name = fields.Str(required=True, validate=Length(min=1))
    date_of_birth = fields.Date("%m/%d/%Y")
    orders = fields.List(fields.Nested(OrderSchema))

    @post_load
    def make_customer(self, data, **kwargs):
        return Customer(**data)

class Customer:
    def __init__(self, first_name, last_name, date_of_birth, orders):
        self.first_name = first_name
        self.last_name = last_name
        self.date_of_birth = date_of_birth
        self.orders = orders

# Example Usage
json_data = {
  'first_name': 'Jane',
  'last_name': 'Smith',
  'date_of_birth': '03/16/1977',
  'orders': [
    {
      'created': '2024-01-25T15:00:04',
      'amount': '100.00'
    }
]}

schema = CustomerSchema()
customer = schema.load(json_data)

print(customer.first_name)  # Output: Jane
print(type(customer.date_of_birth))  # Output: <class 'datetime.date'>
print(type(customer.orders[0].created))  # Output: <class 'datetime.datetime'>
print(type(customer.orders[0].amount))  # Output: <class 'decimal.Decimal'>

Explanation:

  1. Import Necessary Libraries: We import Schema, fields, and post_load from marshmallow, along with specific data types like date, datetime, and Decimal.
  2. Order Schema: We define a OrderSchema to handle the nested order data, specifying created as a DateTime field and amount as a Decimal field with as_string=True to handle the string representation in the JSON.
  3. Customer Schema: We define a CustomerSchema with fields corresponding to the JSON data. We use appropriate field types and validations:
    • first_name and last_name are strings with a minimum length of 1.
    • date_of_birth is a Date field with the format "%m/%d/%Y" to parse the date string correctly.
    • orders is a List field containing nested OrderSchema elements.
  4. post_load Decorator: We use the @post_load decorator to create a Customer instance after successful data loading and validation.
  5. Customer Class: We define a simple Customer class with an __init__ method to initialize its attributes.
  6. Example Usage: We create a CustomerSchema instance and use its load method to deserialize the JSON data into a Customer object.
  7. Output: We demonstrate accessing the attributes of the created Customer object, verifying the correct data types for date_of_birth, created and amount.

Key Points:

  • Marshmallow helps define schemas for data structures, validating and deserializing data into Python objects.
  • Nested schemas like OrderSchema handle complex structures within the JSON data.
  • The post_load decorator allows custom actions after data loading, such as creating instances of your data classes.

Using Marshmallow to Generate Dataclasses

Here's how you can use the marshmallow library in Python to generate dataclasses from your JSON data, handling the specified data types:

from marshmallow import Schema, fields, post_load
from marshmallow_dataclass import dataclass
from datetime import date, datetime
from decimal import Decimal

@dataclass
class Order:
    created: datetime
    amount: Decimal

@dataclass
class Customer:
    first_name: str
    last_name: str
    date_of_birth: date
    orders: list[Order]

class OrderSchema(Schema):
    created = fields.DateTime()
    amount = fields.Decimal()

class CustomerSchema(Schema):
    first_name = fields.Str()
    last_name = fields.Str()
    date_of_birth = fields.Date("%m/%d/%Y")
    orders = fields.List(fields.Nested(OrderSchema))

    @post_load
    def make_customer(self, data, **kwargs):
        return Customer(**data)

# Sample JSON data
json_data = {
  'first_name': 'Jane',
  'last_name': 'Smith',
  'date_of_birth': '03/16/1977',
  'orders': [
    {
      'created': '2024-01-25T15:00:04',
      'amount': '100.00'
    }
]}

# Deserialize JSON data into dataclass instance
schema = CustomerSchema()
customer = schema.load(json_data)

# Access data from dataclass instance
print(customer.first_name)  # Output: Jane
print(customer.date_of_birth)  # Output: 1977-03-16 (date object)
print(customer.orders[0].amount)  # Output: 100.00 (Decimal object)

Explanation:

  1. Import Libraries: Import necessary libraries including marshmallow, marshmallow_dataclass, datetime, and decimal.
  2. Define Dataclasses: Define the Order and Customer dataclasses with the appropriate field types.
  3. Define Schemas: Create marshmallow schemas OrderSchema and CustomerSchema to define how the JSON data should be mapped to the dataclasses.
  4. Field Types: Use specific field types like fields.DateTime, fields.Decimal, and fields.Date with the correct format to ensure proper data conversion.
  5. Nested Schema: Use fields.Nested to handle the list of orders and map them to the Order dataclass.
  6. Post-Load Hook: Define a post_load method in the CustomerSchema to create an instance of the Customer dataclass after deserialization.
  7. Load JSON: Create a CustomerSchema instance and use the load method to deserialize the JSON data into a Customer object.
  8. Access Data: Access the data from the customer object like any other Python object.

Key points:

  • Marshmallow provides a flexible way to define how JSON data should be mapped to Python objects.
  • marshmallow_dataclass simplifies the process by automatically generating schemas from dataclasses.
  • Using specific field types ensures correct data type conversion during deserialization.
  • The post_load hook allows for custom logic after deserialization, such as creating an instance of the dataclass.

Prompt

I need to take this json:

{
'first_name': 'Jane',
'last_name': 'Smith',
'date_of_birth': '03/16/1977',
orders: [
{
'created': '2024-01-25T15:00:04',
'amount': '100.00'
}
]}

And generate some dataclasses.

date_of_birth needs to be a date.
created is a datetime.
amount is a decimal.

How would I do this using the marshmallow library in python?

Response 1.

You can use marshmallow to define your dataclasses and then unmarshal the JSON into instances of those dataclasses:

First, install Marshmallow:

pip install marshmallow

Then, define your dataclasses:

import marshmallow
from marshmallow import fields, post_load
from datetime import datetime, date

class Order(marshmallow.DemoMatch):
    class Meta:
        ordering = ['created']

    id = marshmallow.fields.Int(allow_none=True)
    created = marshmallow.fields.DateTime()
    amount = marshmallow.fields.Decimal(precision=2, scale=2)

    @post_load_to_one('id', load_from_response=True):
    def load_id(self, identifier, data):
        return identifier

class User(marshmallow.DataClass):
    first_name = marshmallow.fields.String()
    last_name = marshmallow.fields.String()
    date_of_birth = marshmallow.fields.Date()
    orders = marshmallow.fields.Nested(Order)

    @post_load
    def load_user(self, data):
        return data

Finally, you can unmarshal the JSON into a User instance:

data = '{ "first_name": "Jane", "last_name": "Smith", "date_of_birth": "1977-03-16", "orders": {"created": "2024-01-25T15:00:04", "amount": "100.00"}}'

import json

user = User().load(json.loads(data))

Now, user will have the expected attributes and instances:

print(user.date_of_birth) # output: datetime.date(1977, 3, 16)
print(user.orders[0].created) # output: datetime.datetime(2024, 1, 25, 15, 04, tzinfo=tzutc())
print(user.orders[0].amount) # output: decimal.Decimal('100.00')

from marshmallow import fields, Schema
from pydantic import BaseModel, positive_float, positive_int

class Order(BaseModel):
    created: positive_int = fields.DateTime(description="Date and time of order creation")
    amount: positive_float = fields.Float(description="Total amount of the order")

class User(BaseModel):
    first_name: str = fields.Str(description="First name of the user")
    last_name: str = fields.Str(description="Last name of the user")
    date_of_birth: datetime = fields.Date(description="Date of birth of the user")

schema = Schema(User)

This code defines two data classes: Order and User. Order contains information about a single order, such as its creation date and amount. User contains information about a user, including their first and last name, date of birth, and a list of orders they have placed.

Marshmallow automatically generates a schema for each of these classes based on their fields. The schema variable at the bottom is a shared instance of the User schema that can be used for validation and serialization.

```python
from marshmallow import fields, Schema, validates_date

class PersonSchema(Schema):
first_name = fields.Str(required=True)
last_name = fields.Str(required=True)

class Meta:
    ordering = ['last_name', 'first_name']

amount = fields.Decimal(required=True)

class Meta:

order = OrderInvoilceSchema().load(json_data['orders'][0])

```

The OrderSchema contains fields for created (a date) and amount (a decimal).
The PersonSchema contains fields for first_name, last_name, and date_of_birth (a date).

The OrderInvoilceSchema contains both person and order data. The person data and order data are loaded separately using the PersonSchema and OrderSchema instances. The order field on the OrderInvoilceSchema is of type fields.Nested(OrderSchema), so we can load the order data into it like this:
